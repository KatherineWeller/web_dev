Local Git Repository Setup for Windows:

1. Git Config Setup

Upon opening Git Bash for Windows the first time, open terminal/shell and type:

$ git config --global user.name "Your name here"
$ git config --global user.email "Your email here"

2. Creating an SSH keypair

If you have not yet created a public/private keypair for your machine, execute the following commands:

$ ssh-keygen -t rsa -C "Your email here"

You may also avoid the use of SSH keys by using the GitHub Desktop.
You will be prompted to save your keypair, press enter to save it in the default location, and press enter twice more to save it with no security passphrase.

3. Add SSH keypair to your GitHub Account

Copy your SSH Public Key into you Github Account Settings:

  • Go to your Github -> Account Settings
  • Click “SSH Keys” on the left menu bar.
  • Click “Add SSH Key”.
  • Add a title (ex: Default_Public_Key) and paste the entire public key into the text box.

To test your connection to the remote host, type:

$ ssh -T git@github.com

If you see the following outcome, your connection was successful.

Hi username! You've successfully authenticated, but Github does
not provide shell access.

4. Creating a Local Repository

You need to cd into the directory where you want to keep your local repository, and run:

$ git init

Within this directory, create a file (usually a README first) and save it to the directory. Then, run the following command to add this file to your list of tracked files:

$ git add .

The . specifies ‘all’ files, but you can select specific ones as well if you please.
To commit these files to Git, run these commands to commit the change along with an intial comment of your choice, and push them to the remomte repository on github.com.

$ git commit -m 'Initial Comment'
$ git push -u origin master

You can check the status of your tracked and untracked files at any time with this command:

$ git status

And that’s the bare bones of how to set up and connect a git repository on your local machine to one on the GitHub website.
 

Common Errors

If you instead recieve the error git@github.com: Permission denied (publickey). you may need to add your key to the SSH authentication agent with the followning commands:

$ eval $(ssh-agent -s)
$ ssh-add ~/.ssh/id_rsa

If the error remote: Permission to KatherineWeller/web_dev.git denied to xilinx-xhub-users-g1. fatal: unable to access 'https://github.com/KatherineWeller/web_dev.git/': The requested URL returned error: 403 is recieved, input this command to remedy:

$ git remote set-url origin git@github.com:MrsColombo/elsevier-twitter-rest-api.git
